AI Resume Analyzer - Project Information File

Project Name:
AI Resume Analyzer

Project Type:
Full-stack web application for resume analysis and tech job matching

Project Goal:
The goal of this project is to help users upload a PDF resume, extract its text, analyze its content, compare it against a selected tech job role, and generate actionable feedback. The application is designed to simulate an ATS-style resume review process with a more modern user experience and richer scoring logic.


1. Core Idea

This project analyzes resumes for technical roles such as Software Engineer, Frontend Developer, Backend Developer, Full Stack Developer, Data Scientist, Machine Learning Engineer, DevOps Engineer, QA Engineer, Data Analyst, and Cybersecurity Analyst.

The system:
- accepts a PDF resume from the user
- extracts text from the PDF
- identifies skills, education, and experience
- checks if important sections are missing
- compares resume content with a structured tech jobs dataset
- calculates a resume score out of 100
- suggests improvements
- supports custom skills and custom keywords
- generates a downloadable text report

3. Main Features

A. Resume Upload
- User uploads a PDF file
- Only PDF files are allowed
- File size limit is enforced in Flask

B. Resume Text Extraction
- Text is extracted from PDF pages using PyPDF2
- The extracted content is normalized before analysis

C. Resume Analysis
- Detects skills from a large internal skill catalog
- Extracts education details
- Extracts experience details
- Detects missing sections such as:
  - Projects
  - Skills
  - Certifications

D. Tech Role Matching
- Matches resume content with a selected job role from a dataset
- Separates:
  - matched required skills
  - missing required skills
  - matched optional skills
  - missing optional skills

E. Resume Scoring
- Generates a score out of 100
- Score is based on:
  - skills match
  - completeness
  - keyword coverage
  - role similarity
  - penalties for weak structure or missing content

F. Suggestions Engine
- Provides suggestions to improve resume quality
- Examples:
  - add missing core skills if relevant
  - include projects section
  - improve keyword alignment
  - use stronger action verbs
  - tailor resume more closely to role language

G. Keyword Optimization
- Highlights important missing keywords
- Tracks keyword coverage percentage

H. Custom Skills and Custom Keywords
- User can add custom skills to check manually
- User can add custom ATS keywords or phrases
- These custom inputs are included in analysis and scoring

I. Download Report
- Generates a downloadable `.txt` report after analysis

J. Modern UI
- Responsive layout
- Drag-and-drop upload area
- Score gauge
- Animated progress bar
- Dark mode toggle
- Cards for result sections
- Clean spacing and polished styling

8. NLP / Matching Logic

This project does not use a heavy deep learning NLP pipeline.
Instead, it uses a practical hybrid approach:

A. Rule-based text analysis
- detect section headers
- detect skills using a skill catalog
- extract education and experience using pattern-based logic

B. Alias-aware matching
- common variants are mapped using regular-expression patterns
- examples:
  - js -> JavaScript
  - sklearn -> Scikit-learn
  - tf -> TensorFlow
  - k8s -> Kubernetes
  - iam -> Identity and Access Management

C. Dataset-driven role intelligence
- each role provides required skills, optional skills, keywords, and related tools

D. ML-assisted similarity
- uses TF-IDF vectorization from scikit-learn
- compares resume text to a target role profile
- produces a role-fit similarity percentage


9. Scoring Logic

The final score is out of 100.

Breakdown:
- Skills Match: 35 points
  - required skills contribute more than optional skills
- Completeness: 25 points
  - based on presence of sections like education, experience, projects, skills, certifications
- Keywords: 20 points
  - based on role keywords and related tools found in the resume
- Role Similarity: 20 points
  - based on TF-IDF semantic similarity

Possible penalties:
- resume too short
- missing important sections
- too many keyword gaps


10. Accuracy Improvements Added

Compared to a very basic resume analyzer, this version is more accurate because:
- it uses a structured role dataset instead of a tiny hardcoded skill list
- it distinguishes core and optional skills
- it supports custom skills and keywords

- it includes TF-IDF similarity scoring
- it includes role-specific keywords and related tools

16. Limitations

Current limitations:
- PDF extraction quality depends on how the PDF was created
- scanned-image PDFs without selectable text may not work well
- the project uses heuristic and TF-IDF methods, not a large language model
- role scoring is only as strong as the current local dataset
- report download is currently text-based, not PDF
